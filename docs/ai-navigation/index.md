# AI 导航索引

这一栏用于把站内知识整理成更适合快速抓取和快速选路的结构。

## 这一栏怎么用

- 如果你已经知道对象名，先看 [对象与能力索引](/ai-navigation/object-capability-index)
- 如果你只有需求描述，先看 [问题到页面映射](/ai-navigation/question-to-page-map)
- 如果你准备开始改代码，先看 [AI 工作顺序](/ai-navigation/ai-workflow)
- 如果你被术语卡住了，先看 [术语别名映射](/ai-navigation/term-aliases)

## 更适合解决

- 进入项目后，不知道应该先读哪个栏目
- 知道某个对象名，但不知道它属于 KScript、内容系统还是远程开发
- 遇到同义词、别名或历史叫法，不确定是不是同一个东西
- 想让 AI 在写代码前，先走一条更稳定的阅读路径

## 常见对象

- `k.request`
- `k.response`
- `k.site.labels`
- `k.site.htmlBlocks`
- `k.content.*`
- `src/Api`
- `src/Module`

## 不要混淆

- 这一栏是导航层，不是对象定义页
- 它负责告诉你先去哪里看，不负责替代原页面正文
